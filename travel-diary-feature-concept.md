# 여행기(Travel Diary) Output 기능 — 컨셉 문서

## 1. 컨셉 요약

여행이 끝난 후 앨범에 쌓인 **일정 · 사진 · 감상문**을 한 편의 **여행기**로 자동 생성해 멤버들이 보관·공유할 수 있게 하는 기능.

> "여행 중에는 흩어진 조각으로 기록하지만, 여행이 끝나면 한 권의 책처럼 정리되어 남는다."

---

## 2. 활용 가능한 기존 데이터

이미 v0.5 스키마에 모인 데이터로 대부분 구성 가능합니다.

| 데이터 | 소스 | 여행기 활용 |
|---|---|---|
| 앨범 메타정보 | `albums` (title, start/end_date, destination) | 표지·여행 정보 |
| 일자별 일정 | `itinerary_items` | 각 챕터(일자)의 뼈대 |
| 사진 | `photos` | 본문 이미지, 표지 후보 |
| 감상문 | `diary_entries` | 각 멤버 시점의 본문 |
| 멤버 | `album_members` | 공동 저자 표시 |
| 좌표 | `albums.destination_lat/lng` | 지도 삽입(선택) |

## 3. Output 형식 후보 (방식 검토용 옵션)

### A안. 정적 웹페이지 (공유 URL) --> 선정 
- 결과물: 읽기 전용 웹 URL (예: `travel-album.app/diary/[token]`)
- 장점: 공유 쉬움, 모바일/PC 어디서나 열림, 멤버 외 가족·친구에게도 보여줄 수 있음
- 단점: 영구 보관성 약함(서비스 종료 시 사라짐)
- 구현: 기존 데이터 그대로, SSG(Static Site Generation) 한 페이지 추가

### B안. PDF 다운로드
- 결과물: A4/모바일 PDF 파일
- 장점: 영구 보관, 출력 가능, 오프라인 열람
- 단점: 동영상·인터랙션 불가, 디자인 제약, 생성 비용(서버사이드 렌더링)
- 구현: Puppeteer 또는 React-PDF 사용

### C안. 매거진 스타일 인터랙티브 페이지
- 결과물: 스크롤 인터랙션·애니메이션이 있는 웹페이지
- 장점: 가장 감성적, 공유 시 임팩트 큼
- 단점: 디자인·구현 비용 가장 큼
- 구현: 풀스크롤 레이아웃 + 모션 라이브러리(Framer Motion 등)

### D안. 포토북 인쇄 연동 (외부 서비스)
- 결과물: 실제 종이책 (Snapfish, 카카오메이커스 등 연동)
- 장점: 실물 보관, 선물 가능
- 단점: 외부 결제·배송 연동 복잡, 사업 모델 필요
- 구현: 3rd-party API 연동 (현실적으로 매우 후순위)

> **PM 추천 우선순위**: A안(웹페이지) → B안(PDF) → C안(매거진) → D안(인쇄)
> A안부터 시작하면 빠르게 만들고 사용자 반응 보면서 B/C로 확장 가능

## 4. 구조 시안 (공통)

### 표지
- 앨범 제목 / 여행 기간 / 목적지
- 대표 사진(커버 또는 멤버가 직접 선정)
- 참여 멤버 닉네임 (공동 저자처럼)

### 본문 (일자별 챕터)
각 날짜마다:
- 날짜 헤더 + 일정 요약 (장소·시간 리스트)
- 그날의 사진들 (그리드 또는 일렬 배치)
- 멤버별 감상문 (이름 + 작성 시간 + 내용)
- 날씨 정보 (있다면)

### 마무리
- 여행 통계 (총 사진 수, 방문 장소 수, 멤버별 작성 글 수)
- 멤버별 갤러리(선택)

## 5. 결정이 필요한 정책 질문

이 기능 구체화 전에 답해야 할 것들:

| # | 질문 | 옵션 | 결정 |
|---|---|---|---|
| Q1 | 언제 생성되는가? | (a) 여행 종료일 자동 생성 / (b) 방장이 수동 트리거 / (c) 멤버 누구나 가능 | c |
| Q2 | 사진은 전부 들어가는가? | (a) 전부 자동 / (b) 멤버가 하이라이트 선정 / (c) AI 자동 큐레이션 | a |
| Q3 | 누가 편집할 수 있는가? | 표지·순서·캡션 등 — 방장만 / 전체 / 본인 콘텐츠만 | 누구나 |
| Q4 | 공유 범위는? | 멤버만 / 링크 가진 사람 누구나 / 검색 가능 공개 | 링크를 가진 누구나 |
| Q5 | 한 앨범당 여행기 개수는? | 1개(최신본 덮어쓰기) / 여러 버전 보관 | 1개 |
| Q6 | 멤버 콘텐츠 권리 | 강퇴/탈퇴한 멤버의 콘텐츠 포함 여부 | 포함 하지 않음 |

  created_at, updated_at
